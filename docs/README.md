# CommandLayer Protocol — Commercial (v1.0.0 Candidate)

*The commerce verb layer for agents. x402‑ready. ENS/8004 discoverable.*

## What’s inside
- 10 **Commercial verbs** (requests + receipts) suitable for finance, identity, auth, payments, logistics.
- **Aliases**: one per verb for adoption, never superseding canonicals.
- **C2‑Pro License**: elastic tiering with clear triggers for paid use.

### Target verbs (initial 10)
- checkout, invoice, refund, verify, sign,
- subscribe, ship, track, disburse, settle

See **SPEC.md** for shapes and invariants.

## Migration: Commons → Commercial
Use **Commons** for universal messaging and data ops. Move to **Commercial** when your agent performs regulated, financial, identity, authentication, settlement, or transactional actions.  
**Rule of thumb:** Commons = communication. Commercial = commerce.

## Interop
- **x402** invocation compatible
- **ERC‑8004** discovery via ENS/registry
- Works with SDKs, routers, and gateways

## Licensing (C2‑Pro overview)
- Free for evaluation and small projects
- Triggered commercial terms when either:
  - **Revenue ≥ $50k/year**, or
  - **Org size ≥ 20 employees**, or
  - **≥ 1M agent invocations/month**
- See **EULA-C2-PRO.md** and **PRICING.md**
